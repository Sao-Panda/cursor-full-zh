## 2. 汉化双层架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Cursor 工作台 (workbench.html)                │
├──────────────────────────────┬──────────────────────────────────────┤
│  L1 官方语言包层              │  L2 注入脚本层 (Cursor_Localization.js) │
│  VSCode-language-pack-zh-hans │  词典 + DOM 运行时替换                  │
├──────────────────────────────┼──────────────────────────────────────┤
│  • 顶部菜单栏                 │  • Cursor 设置侧栏（账户/模型/规则/MCP）  │
│  • VS Code 标准设置项         │  • Agent / Composer 对话区             │
│  • 内置扩展 NLS               │  • 插件市场、欢迎页、Git 审查面板         │
│  • locale.json → zh-cn        │  • 通知 Toast、浏览器内嵌页、设计面板     │
└──────────────────────────────┴──────────────────────────────────────┘
```

| 层级 | 机制 | 不覆盖范围 |
|------|------|------------|
| L1 语言包 | 安装 VSIX + `locale.json` | Cursor 私有 React/Web 组件文案 |
| L2 注入脚本 | MutationObserver + 作用域补丁 | 用户代码、Agent 生成内容、Monaco 编辑器正文 |

## 5. 运行时模块组装

```
bootstrap.js   → 入口、localStorage 迁移、词典块占位符
engine.js      → 翻译引擎、XiuZheng_* 作用域补丁（约 50 个函数）
helpers.js     → FanYi_Scope_*、FanYi_Cursor_SheZhi_*、FanYi_ZiShu_QuYu
market.js      → 插件市场懒加载与描述翻译
keywords.js    → QuanJu_GuanJianCi_Biao 门闩关键词（按场景触发补丁）
init.js        → MutationObserver 初始化与延迟全量修正
```

**翻译查找顺序**（`ChaZhao_FanYi`）：

1. 精确匹配 `FanYi_CiDian`（由 Core + Ad 合并）
2. 正则 `MoShi_FanYi`（Pattern_Dictionary）
3. 部分片段 `TiHuan_BuFen_WenBen`（Partial + Dropdown + 设置片段）

**跳过区域**（不修改用户内容）：

- `.monaco-editor .view-lines`（编辑器正文）
- `TEXTAREA` / `INPUT` 用户输入（placeholder 除外）
- 设置 key 形如 `foo.bar.baz` 的纯标识符

## 6. 维护工作流

### 6.1 定位未翻译英文

1. **设置页原文查找**（需 Node.js）：

   ```bash
   node Cursor_Setting_Lookup.js --text "英文片段"
   node Cursor_Setting_Lookup.js --key "setting.key"
   ```

2. **判断归属层级**：对照本文档 §3，确定应写入哪个 JSON。

3. **选择词典类型**：

   | 场景 | 编辑文件 | 格式 |
   |------|----------|------|
   | 固定 UI 文案 | `Core_Dictionary.json` 对应 section | `["English", "中文"]` |
   | 含数字/变量 | `Pattern_Dictionary.json` | `{regex, flags, replacement}` |
   | 长句片段 | `Partial_Fragments.json` | `[["片段", "译文"], ...]` |
   | 下拉选项 | `Dropdown_Fragments.json` | 同上 |
   | MCP/域名说明 | `Cursor_Settings_Fragments.json` | `mcpEntries` / `domainEntries` |
   | 左下角 Toast | `Ad_Popup_Dictionary.json` | `entries` |
   | 插件市场 | `Plugin_Marketplace_Dictionary.json` | `pluginNames` 等 |

4. **校验与注入**：

   ```bash
   python localization/tools/validate_dictionary.py
   python Cursor_Localization_Tool.py
   ```

5. **完全退出并重启** Cursor。

### 6.2 版本号更新规范

| 变更类型 | 更新项 |
|----------|--------|
| 新增/修改词条 | 对应 JSON 的 `updatedAt`（`YYYY-MM-DD`） |
| 新增 section 或层级结构调整 | 本文档 `文档版本` minor +1 |
| 语言包 VSIX 更换 | 更新 §1 版本对照表中的 VSIX 版本号 |
| 重大架构变更（如新补丁层） | 本文档 major +1 |

### 6.3 注意事项

- 中文译文中**不要使用全角引号** `""`，以免生成的 JS 字符串报错。
- Agent **对话内容**与**模型输出**不属于界面汉化范围。
- Cursor 更新后 `workbench.html` 可能被覆盖，需重新运行汉化脚本。

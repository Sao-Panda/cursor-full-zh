# Cursor 完整汉化工具分享：一键把界面变成中文

> **平台说明**：本仓库的汉化工具**仅支持 macOS**（`Cursor.app`）。暂不提供 Windows / Linux 版本。  
> **适配版本**：Cursor `3.8.23`（内置 VS Code `1.105.1`，验证于 2026-06-24）；`3.8.11` 亦验证通过，同系列 `3.8.x` 通常可直接复用。  
> 本工具面向日常使用 Cursor 的 **Mac 用户**，在不修改 Cursor 核心程序的前提下，将界面中的英文菜单、按钮、提示和说明替换为中文，降低阅读和操作成本。已实现 编辑器模式（Editor）和 智能体模式（Agent）两种界面的全面汉化。

---

## 工具介绍

Cursor 官方目前尚未提供中文界面或汉化包，常见做法是安装VSCode汉化插件实现部分汉化，菜单、通用设置等会变成中文，但智能体侧栏、Cursor 设置、插件市场等 **Cursor 专有界面** 往往仍是英文，给习惯使用中文界面的用户增加了学习和使用成本。

**Cursor 完整汉化工具** 的做法是在自动安装VSCode汉化包的同时，向工作台注入带词典的运行时脚本，把**Cursor专有界面**也替换成中文，让日常使用的界面尽量一眼能读懂。工具已实现编辑器模式（Editor）和智能体模式（Agent）两种界面的全面汉化。

https://www.vibepm.net/cursor/


### 覆盖区域：

- **设置页**：账户、模型、规则、工具与 MCP、隐私等选项
- **智能体窗口**：对话、命令面板、模式切换（计划 / 询问 / 调试 / 多任务）等
- **插件市场**：插件名称、简介、获取按钮及市场相关文案
- **其它界面**：欢迎页、Git 变更面板、部分通知与推广弹窗等

### 主要特点

- **自动安装VSCode汉化包**：一键自动安装VSCode汉化扩展（覆盖标准菜单、通用设置等），如有更新，将自动根据版本下载匹配的语言包并安装，省去自己去扩展市场手动搜索、安装语言包的步骤。
- **双层汉化，覆盖面更全**：通过注入脚本 + 本地词典，完整翻译 Cursor 专有界面，例如智能体侧栏、Cursor 设置（账户 / 模型 / 规则 / MCP）、插件市场、欢迎页、Git 面板及部分通知弹窗等。
- **不改动核心程序，可一键还原**：只修改工作台相关文件（如 `workbench.html`），需要英文时运行「取消汉化」脚本即可恢复，已装的VsCode**汉化包**可保留。
- **维护成本低**：Cursor 小版本更新后若有个别位置变回英文，再运行一次启动脚本即可，一般无需重装软件。
- **其它说明**：日常点得到的界面大多已是中文；个别菜单、动态文案或新版本新增词条可能仍为英文，可在词典中补充后重新注入。Agent 对话与模型生成内容不属于界面汉化范围。

## 如何使用

### 环境要求

- **操作系统**：macOS（需已安装 `Cursor.app`）
- **Python**：3.8+

1. 下载并解压汉化工具（内含 `启动汉化_Mac.sh` / `取消汉化_Mac.sh`）。
2. 在终端执行 `bash 启动汉化_Mac.sh`。
3. 脚本会安装语言包并注入汉化，按提示可选打开 Cursor。
4. **完全退出并重启** Cursor（不要只重载窗口）。

**Mac在终端中运行汉化脚本：**
1. 打开**终端**：按 `Cmd + 空格`，输入「终端」后回车。
2. 在终端输入 `bash`加空格，再把 `启动汉化_Mac.sh` 拖入窗口补全路径后回车，输入`Y`进行汉化。

**若安装后顶部菜单仍为英文**，请手动切换显示语言：
1. 按 `Cmd+Shift+P` 打开**命令面板**
2. 输入 `Configure Display Language`，选择 **配置显示语言**
3. 在列表中选择 **中文(简体)**（`zh-cn`）
4. 按提示**完全退出并重启** Cursor

---

## 恢复英文

在终端运行 `取消汉化_Mac.sh`（或 `bash 取消汉化_Mac.sh`）；

同样要完全退出并重启。语言包会保留，只是撤销注入、恢复备份文件。

## 常见问题

- **运行脚本报错 / 汉化无输出**：确认已安装 **Python 3.8+**，终端执行 `python3 --version` 验证。
- **更新 Cursor 后汉化失效了**：重新再运行一次汉化脚本即可。
- **已经执行汉化但是没有生效**：确认已完全重启了 Cursor。
- **汉化后顶部菜单仍是英文**：按 `Cmd+Shift+P` → 输入「Configure Display Language」→ 选 **中文(简体) / zh-cn** → 再完全重启一次。如果没有 **中文(简体)** 选项，在扩展市场中搜索并安装 **Chinese (Simplified) Language Pack** 后重新设置。
- **执行脚本找不到 Cursor**：一般是因为 Cursor 未装在默认目录。可在运行脚本前设置环境变量，或在 `Cursor_Localization_Tool.py` 顶部用户配置区域填写路径：

```bash
export CURSOR_INSTALL_DIR="/你的路径/Cursor.app"
export CURSOR_USER_DATA_DIR="$HOME/Library/Application Support/Cursor"
```

保存后重新运行启动脚本即可。

## 注意事项

1. **每次升级 Cursor 后重跑一次脚本**：升级会覆盖安装目录，汉化会失效，重跑即可恢复。
2. **完全退出再重启**：用 `Cmd+Q` 退出（活动监视器确认无 `Cursor` / `CursorUIViewService` 进程残留），不要只关窗口。
3. **不要和「直接改主程序 JS」的汉化工具混用**：那类工具会触发 “Your Cursor installation appears to be corrupt. Please reinstall.” 提示。本工具只注入 `workbench.html` 并同步 `product.json` 校验值，不改主 JS，不会触发该提示；若已被其他工具改坏，先 `--restore` 或重装 Cursor 再用本工具。
4. **两层汉化**：标准菜单是微软官方语言包（可持续更新）；Cursor 专有界面靠本地词典（社区维护，新词条可能滞后）。
5. **Agent 对话内容不汉化**：只翻译界面，不翻译模型生成的内容。
